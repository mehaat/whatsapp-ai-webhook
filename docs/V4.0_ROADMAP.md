# ME-HAAT Fashion AI Bot — v4.0 Enterprise Roadmap

Phase 1 (shipped — see `V4.0_RELEASE_NOTES.md`) delivered the live product-card
flow and the enterprise **foundations**: opt-in SQLAlchemy persistence,
structured logging + trace ids, security hardening (PII masking, optional token
encryption, security headers), and health/readiness/liveness endpoints.

The remaining v4.0 scope from the enterprise brief is intentionally sequenced
into safe, incremental phases so the **live production bot is never broken by a
big-bang rewrite**. Each phase is independently deployable and backward
compatible.

## Phase 2 — Persistence cutover (low risk)
- Turn on `USE_DATABASE=true` in staging; back the conversation memory and the
  Shopify token store with the repositories already written (interfaces already
  match). Keep the in-memory store as an automatic fallback.
- Add `OrderCache`, `FAQ`, `Knowledge`, `Session` models + repositories.
- Add Alembic migrations (replace `create_all`).

## Phase 3 — Shopify Admin API v4 depth
- Add a GraphQL Admin API client alongside the REST client (query builder,
  automatic retry, cursor pagination, rate-limit/cost handling).
- Extend coverage: collections, customers, draft orders, fulfillments, discount
  codes, abandoned checkouts, store info.
- Add `/shopify/uninstall` + `/shopify/webhooks` (app/uninstalled, GDPR topics)
  with HMAC validation (helpers already exist in `utils/security.py`).
- Offline + optional online tokens; encrypted token storage on by default;
  token rotation hooks.

## Phase 4 — AI architecture as services
- Split the AI layer into Prompt Manager, Conversation Manager, Memory Manager,
  Knowledge Manager, Context Builder, Gemini Service.
- Add a tool-calling layer exposing verified Shopify tools (search, recommend,
  similar products, inventory, order tracking, draft order, collections,
  policies, delivery estimate, customer lookup).
- Intent detection, conversation compression, token optimisation, prompt
  versioning, and streaming-ready responses.

## Phase 5 — Knowledge layer
- Structured knowledge: policies, shipping, returns, exchange, collections.
- Ingestion: PDF/CSV/Excel/Google Sheets/website crawl.
- Embeddings + vector store for semantic FAQ/product retrieval.

## Phase 6 — WhatsApp services
- Reusable message services: text, interactive buttons, lists, templates,
  catalog, media (image/doc/video), location, contacts, reactions, typing
  indicator, read receipts, delivery status.
- Retry queue + dead-letter queue for outbound messages.

## Phase 7 — Hardening, delivery & docs
- Full test matrix (OAuth, webhook, WhatsApp, Gemini, repositories, security).
- Docker + Gunicorn tuning, graceful shutdown, startup validation.
- Architecture diagram, API docs, security docs, migration + rollback guides,
  production checklist.

---

### Suggested execution model
Run each phase as its own change set with its own verification checklist and
deploy. Keep `USE_DATABASE` and native-catalog behind flags until validated in
staging. This preserves the "never break production" guarantee while steadily
reaching the full enterprise target.
