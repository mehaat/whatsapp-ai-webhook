# ME-HAAT Fashion AI Bot — v4.0 Release Notes (Phase 1)

**Status:** Production-ready, fully backward compatible.
**Scope of this release:** The headline customer-facing fix (live Shopify
product cards) plus the enterprise foundations (persistence, structured
logging, security hardening, health/monitoring). The remaining v4.0 items
(GraphQL Admin API, AI tool-calling, vector knowledge base, full test matrix,
Docker) are sequenced in `docs/V4.0_ROADMAP.md`.

> **Guiding rule honoured:** nothing that worked in v3.0 was removed or changed
> in a breaking way. Every new capability is additive and, where it needs new
> infrastructure (database, token encryption, native catalog), it is **opt-in**
> and degrades gracefully when unconfigured.

---

## 1. The bug this release fixes

When a customer sent **"Show saree"**, the bot replied with the static
catalogue link instead of live Shopify products.

**Root cause (confirmed in code):** in `app.py`, catalogue intent was checked
*before* product search, and `ai/faq.py`'s catalogue keywords include `saree`,
`sarees`, `products`, `show me`, etc. So `wants_catalogue("Show saree")`
returned `True`, the bot sent the catalogue link and returned — the product
search branch and `send_product_card()` were never reached.

**The fix:** product-search intent is now detected and handled **before** the
catalogue link. When products are found they are sent as WhatsApp product
cards immediately; the catalogue link is used only as a fallback when no
products are found or Shopify is unavailable.

## 2. File-by-file changelog

### Modified

- **`app.py`** — Reworked `handle_customer_message` precedence: (1) pagination
  "more/next", (2) greeting, (3) order status, (4) **product-search intent →
  live Shopify search → product cards → optional short Gemini recommendation**,
  (5) catalogue link *fallback only*, (6) FAQ+Gemini default. Added
  `_handle_product_search`, `_send_next_product_page`, `_is_more_request`,
  best-effort AI logging, extended health endpoints (`/health`, `/health/live`,
  `/health/ready`), security-headers middleware registration, DB bootstrap,
  version → 4.0. All previous helpers (`_build_order_status_context`,
  `_build_product_search_context`, `generate_ai_reply`, `_finalize_reply`)
  retained.

- **`shopify/search.py`** — Added `ProductMatch.short_description`, `.handle`,
  `.variant_count`, `.stock_label`, `.to_card_dict()`, and a URL in
  `.to_context_line()`. Added `detect_product_search_intent()`,
  `search_and_rank()` (combines title + tags + product type + filters, relevance
  ranking, de-duplication, capped results), `currency_symbol()`, HTML→text
  description parsing, and a broader, more robust `extract_search_filters()`
  (ranges, "upto/within", more colours/fabrics/occasions). All v3.0 functions
  (`search_products`, `get_product_details`, `list_collections`,
  `get_products_in_collection`, `select_variant`) unchanged.

- **`whatsapp/sender.py`** — Upgraded `send_product_card()` to render Title,
  Price, Currency, Availability, Category, Variant count, Short description and
  Product URL (max 5, de-duplicated, nicely formatted). Added
  `send_catalog_product_message()` (native WhatsApp `product_list`) and
  `send_products()` — a dispatcher that uses the native Meta Commerce catalog
  when `WHATSAPP_CATALOG_ID` is set and falls back to text cards otherwise. All
  existing send functions unchanged.

- **`memory/store.py`** — Added product-search pagination: `set_last_search`,
  `get_next_search_page`, `has_active_search`, `clear_search`. Search state
  expires with the conversation. All existing methods unchanged.

- **`config.py`** — Added optional settings: `whatsapp_catalog_id`,
  `product_reco_enabled`, `token_encryption_key`, `use_database`,
  `database_url`, `log_format`, `log_level`, and a `version` property
  (`APP_VERSION = "4.0"`). All new settings have safe defaults; required-var
  validation unchanged.

- **`utils/logging.py`** — Added optional structured JSON logging
  (`LOG_FORMAT=json`) and a per-request trace id
  (`new_request_id`/`bind_request_context`/`clear_request_context`). Default
  text format is byte-for-byte the v3.0 format. `logger`, `redact`,
  `is_sensitive_key`, `log_execution_time` unchanged.

- **`utils/security.py`** — Added `mask_pii()` (masks phones/emails before
  logging/storage) and optional token encryption at rest
  (`encrypt_token`/`decrypt_token`/`generate_encryption_key`, Fernet-based,
  no-op passthrough when unconfigured). All existing helpers unchanged.

- **`requirements.txt`**, **`render.yaml`**, **`.env.example`** — Added the new
  optional dependencies and environment variables (see §3). Existing entries
  unchanged.

### Added

## 3. New environment variables (all optional)

| Variable | Default | Purpose |
|---|---|---|
| `WHATSAPP_CATALOG_ID` | *(blank)* | Meta Commerce catalog id → native WhatsApp product messages. Blank = text cards. |
| `PRODUCT_RECO_ENABLED` | `true` | Send a short Gemini recommendation after product cards. `false` = cards only. |
| `LOG_FORMAT` | `text` | `text` (v3.0 format) or `json` (structured logs). |
| `LOG_LEVEL` | `INFO` | Standard Python log level. |
| `USE_DATABASE` | `false` | Enable the SQLAlchemy persistence layer. |
| `DATABASE_URL` | `sqlite:///mehaat.db` | DB URL (SQLite / Postgres) — used only when `USE_DATABASE=true`. |
| `TOKEN_ENCRYPTION_KEY` | *(blank)* | Fernet key to encrypt Shopify tokens at rest. Blank = plaintext (v3.0). |

Generate a token-encryption key with:

```
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

## 4. Production deployment instructions (Render)

1. Merge/pull this branch into the deploy branch.
2. `pip install -r requirements.txt` runs automatically on Render (now includes
   `SQLAlchemy` and `cryptography`).
3. No new **required** env vars — the app deploys unchanged. Optionally set any
   of the v4.0 variables above (already listed in `render.yaml`).
4. Deploy. Render health check continues to hit `/health` (now returns
   `version: "4.0"` plus a `components` block).
5. Verify with the checklist in §5.

**If you want native WhatsApp product messages:** connect a Meta Commerce
catalog, then set `WHATSAPP_CATALOG_ID`. Until then, rich text cards are sent.

**If you want persistence:** set `USE_DATABASE=true`. On the free/starter tier
the SQLite file lives on the mounted disk; for multi-worker durability point
`DATABASE_URL` at Render Managed Postgres and uncomment `psycopg2-binary` in
`requirements.txt`.

## 7. Rollback strategy

This release is a single commit on top of v3.0 with **no schema migration in
the default path** (`USE_DATABASE=false`). To roll back:

```
git revert <this-commit>      # or: git checkout <previous-tag> -- .
# redeploy on Render
```

Because the token store format, OAuth, and webhook are untouched, rolling back
requires no data changes. If you enabled `USE_DATABASE=true`, the created
tables are additive and can be left in place or dropped independently.
