ME-HAAT Fashion Commerce API

Programmatic access to orders, tracking and payment webhooks for the ME-HAAT Fashion AI Bot. JSON over HTTPS, authenticated with a Bearer JWT or an API key.

Interactive API Docs (Swagger) OpenAPI Spec (JSON)

Base URL

All endpoints are served relative to your deployment host:

https://YOUR-HOST/

Authentication

Protected endpoints accept either of two credentials. Public endpoints (order tracking and payment webhooks) require none.

1 · Bearer JWT

Exchange admin credentials for a short-lived HS256 token via POST /api/token, then send it as a bearer token.

Authorization: Bearer <jwt>

2 · API Key

Send a key issued from the admin dashboard in the X-API-Key header. Keys look like mh_live_….

X-API-Key: mh_live_<prefix>_<secret>

Quick start

Fetch a token, then list orders. Or skip the token and use an API key directly:

# Option A — Bearer JWT
TOKEN=$(curl -s -X POST https://YOUR-HOST/api/token \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"••••••"}' | jq -r .token)

curl https://YOUR-HOST/orders \
  -H "Authorization: Bearer $TOKEN"

# Option B — API key
curl https://YOUR-HOST/orders \
  -H "X-API-Key: mh_live_abcd1234_your-secret-here"

Rate limits

Each API key carries its own per-minute request budget (default 120 requests/minute), enforced with a sliding one-minute window. Exceeding it returns 429 Too Many Requests; retry after the window rolls forward. JWT and admin-session traffic are not key-limited.
