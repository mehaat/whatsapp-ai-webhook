ME-HAAT Fashion Commerce API

Programmatic access to orders, tracking and payment webhooks for the ME-HAAT Fashion AI Bot. JSON over HTTPS, authenticated with a Bearer JWT or an X-API-Key.

Interactive API Docs (Swagger) OpenAPI Spec (JSON)

Base URL

All endpoints are served relative to your deployment host:

https://YOUR-HOST/

Replace YOUR-HOST throughout this page with your actual deployment domain.

Authentication

Protected endpoints accept either of two credentials. Public endpoints (order tracking, payment webhooks and token issuance) require none.

1 · Bearer JWT

Exchange admin credentials for a short-lived HS256 token via POST /api/token, then send it as a bearer token.

Authorization: Bearer <jwt>

2 · API Key

Send a key issued from the admin dashboard in the X-API-Key header. Keys look like mh_live_….

X-API-Key: mh_live_<prefix>_<secret>

Rate limits

Each API key carries its own per-minute request budget (default 120 requests/minute), enforced with a sliding one-minute window. Exceeding it returns 429 Too Many Requests with body {"error":"rate_limited"}; retry once the window rolls forward. JWT and admin-session traffic are not key-limited.

Code samples

Pick your language. Each example gets a token, lists orders with it, and tracks an order.

# 1) Get a bearer token
TOKEN=$(curl -s -X POST https://YOUR-HOST/api/token \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"••••••"}' | jq -r .token)

# 2) List orders with the token
curl https://YOUR-HOST/orders?status=shipped&limit=20 \
  -H "Authorization: Bearer $TOKEN"

# ...or use an API key directly instead of a token:
curl https://YOUR-HOST/orders \
  -H "X-API-Key: mh_live_abcd1234_your-secret-here"

# 3) Track an order (public — no auth)
curl https://YOUR-HOST/tracking/MH-2026-0001
import requests

BASE = "https://YOUR-HOST"

# 1) Get a bearer token
resp = requests.post(f"{BASE}/api/token", json={
    "username": "admin",
    "password": "••••••",
})
token = resp.json()["token"]

# 2) List orders with the token
orders = requests.get(
    f"{BASE}/orders",
    params={"status": "shipped", "limit": 20},
    headers={"Authorization": f"Bearer {token}"},
).json()
print(orders["count"], "orders")

# ...or authenticate with an API key instead:
# headers = {"X-API-Key": "mh_live_abcd1234_your-secret-here"}

# 3) Track an order (public — no auth)
tracking = requests.get(f"{BASE}/tracking/MH-2026-0001").json()
print(tracking["status"], tracking["stages"])
const BASE = "https://YOUR-HOST";

// 1) Get a bearer token
const tokenResp = await fetch(`${BASE}/api/token`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ username: "admin", password: "••••••" }),
});
const { token } = await tokenResp.json();

// 2) List orders with the token
const orders = await fetch(`${BASE}/orders?status=shipped&limit=20`, {
  headers: { Authorization: `Bearer ${token}` },
}).then((r) => r.json());
console.log(orders.count, "orders");

// ...or authenticate with an API key instead:
// headers: { "X-API-Key": "mh_live_abcd1234_your-secret-here" }

// 3) Track an order (public — no auth)
const tracking = await fetch(`${BASE}/tracking/MH-2026-0001`).then((r) => r.json());
console.log(tracking.status, tracking.stages);

Webhooks

Payment providers deliver events to a public, per-provider sink. Providers cannot present our JWT, so the endpoint is unauthenticated — each provider adapter verifies its own signature. The endpoint always responds 200 (even on verification failure) to prevent provider retry storms.

POST /payments/webhook/{provider}

provider is one of razorpay, stripe, cashfree, phonepe, manual_upi. Send the provider's raw payload and signature headers; a paid event flips the order to paid.

# Example acknowledgement body
{ "ok": true, "status": "paid" }
